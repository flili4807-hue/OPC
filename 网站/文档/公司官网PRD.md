# 公司官网 PRD

版本：V1.0  
日期：2026-07-13  
适用对象：产品、设计、前端开发、内容运营、AI 开发助手  
项目类型：公司官网 / 品牌展示站 / 商务转化型 Landing Page

## 1. 项目背景

当前公司需要建设一个简洁、专业、具备商务转化能力的官网，用于对外展示品牌定位、核心服务、代表案例与合作入口。

官网应避免传统企业站信息堆砌，重点传达“专业可信、方法清晰、结果导向”的品牌感知。当前视觉方向已确定为“现代科技 × 极简商务”，适合 AI 产品、数字战略、增长咨询、产品设计、创新服务类公司使用。

## 2. 项目目标

### 2.1 业务目标

1. 建立公司对外统一品牌入口。
2. 清晰说明公司能解决什么问题、服务什么客户、提供什么能力。
3. 提升潜在客户对公司专业度与可信度的判断效率。
4. 引导潜在客户通过邮箱、表单或其他联系方式发起咨询。
5. 为后续内容营销、案例沉淀、SEO 和投放转化提供基础页面。

### 2.2 用户目标

访问者进入官网后，应能在 30 秒内理解：

1. 这家公司是谁。
2. 主要服务什么客户。
3. 能提供什么价值。
4. 是否有相关案例或成果证明。
5. 如何进一步联系合作。

## 3. 目标用户

### 3.1 核心用户

1. 潜在企业客户  
   关注服务能力、案例结果、合作方式和可信度。

2. 创业者 / 创新业务负责人  
   关注是否能帮助其从想法、战略、产品到落地。

3. AI 产品 / 数字化项目负责人  
   关注是否具备 AI 场景识别、产品定义、流程自动化和交付能力。

### 3.2 次级用户

1. 合作伙伴  
   关注公司定位、能力边界和合作空间。

2. 投资人 / 媒体 / 招聘候选人  
   关注公司方向、专业形象、品牌可信度。

## 4. 产品定位

公司官网定位为“品牌展示 + 能力说明 + 商务转化”的单页官网。

当前阶段不做复杂 CMS、不做会员系统、不做后台管理。优先交付一个稳定、轻量、视觉统一、可快速上线和迭代的官网首页。

## 5. 范围说明

### 5.1 本期范围

本期建设一个响应式单页官网，包含以下模块：

1. 顶部导航
2. 首屏 Hero
3. 品牌背书 / 能力标签
4. 服务能力
5. 案例展示
6. 品牌原则 / 关于我们
7. 联系合作
8. 页脚

### 5.2 暂不包含

1. 后台管理系统
2. 博客 / 文章 CMS
3. 多语言切换
4. 用户登录注册
5. 在线支付
6. 复杂动画系统
7. CRM 自动化集成
8. 在线预约日历

以上能力可作为后续版本规划。

## 6. 页面结构

### 6.1 首页

首页为单页滚动结构，通过顶部导航锚点跳转到对应区域。

页面顺序如下：

1. 顶部导航
2. Hero 首屏
3. 能力 / 品牌标签
4. 服务能力
5. 代表案例
6. 关于 / 原则
7. 联系合作
8. 页脚

## 7. 功能需求

### 7.1 顶部导航

#### 功能说明

顶部导航用于帮助用户快速理解网站结构，并跳转到关键区域。

#### 内容要求

1. 左侧展示品牌 Logo 和品牌名。
2. 中间展示导航入口：
   - 能力
   - 案例
   - 关于
3. 右侧展示主要行动按钮：
   - 开始合作

#### 交互要求

1. 点击品牌 Logo 返回页面顶部。
2. 点击导航项平滑跳转到对应模块。
3. 点击“开始合作”跳转到联系模块。
4. 移动端需保持可读性和可点击性。

#### 验收标准

1. 所有导航链接可正常跳转。
2. 首屏及滚动状态下导航清晰可见。
3. 移动端无内容挤压或横向滚动。

### 7.2 Hero 首屏

#### 功能说明

Hero 是用户进入官网后的第一认知区域，需要明确传达品牌定位和核心价值。

#### 内容要求

1. 品类标签，例如：DIGITAL STRATEGY · AI PRODUCT。
2. 主标题，例如：把复杂，变成清晰的增长。
3. 简短价值描述。
4. 两个行动入口：
   - 查看案例
   - 了解服务
5. 右侧展示抽象数据或增长视觉。

#### 交互要求

1. 点击“查看案例”跳转到案例模块。
2. 点击“了解服务”跳转到服务模块。
3. 首屏视觉在桌面端左右分栏，移动端上下排列。

#### 验收标准

1. 用户无需滚动即可看到品牌名、核心价值和主 CTA。
2. 桌面端首屏视觉完整。
3. 移动端主标题、按钮、视觉元素不遮挡。

### 7.3 品牌背书 / 能力标签

#### 功能说明

用于快速强化公司服务方向，降低用户理解成本。

#### 内容要求

展示一句说明文案和 3-5 个能力标签，例如：

1. 战略
2. 产品
3. 品牌
4. AI

#### 验收标准

1. 标签简洁明确。
2. 与后续服务模块内容一致。

### 7.4 服务能力模块

#### 功能说明

展示公司核心服务能力，让潜在客户判断是否匹配需求。

#### 内容要求

建议展示 3 个核心服务：

1. 战略与定位  
   包含市场洞察、价值主张、增长路径与行动优先级。

2. 产品与体验  
   包含产品定义、用户旅程、原型设计与体验系统。

3. AI 与自动化  
   包含场景识别、Agent 方案、流程重构与效果评测。

#### 交互要求

1. 每个服务项可支持后续扩展为详情页。
2. 当前版本可仅展示静态内容。

#### 验收标准

1. 每项服务都有标题和一句说明。
2. 服务内容不超过用户快速阅读负担。
3. 桌面端排列规整，移动端纵向排列。

### 7.5 案例展示模块

#### 功能说明

通过代表案例证明公司方法和结果，提升可信度。

#### 内容要求

每个案例包含：

1. 案例类型
2. 年份
3. 案例标题
4. 简短说明
5. 结果指标

示例案例：

1. AI Product  
   主题：让专业服务拥有数字分身  
   结果：交付效率提升 3.2×

2. Growth System  
   主题：重构从流量到成交的路径  
   结果：有效线索增长 +47%

#### 注意事项

如果案例数据尚未真实验证，正式上线前应替换为真实客户案例、内部项目案例，或明确标注为示例。

#### 验收标准

1. 至少展示 2 个案例。
2. 每个案例都有结果表达。
3. 结果数据必须可解释、可追溯，避免夸大宣传。

### 7.6 关于 / 品牌原则模块

#### 功能说明

用于表达公司方法论和价值观，强化品牌记忆点。

#### 内容要求

1. 一句核心原则。
2. 3 个关键词，例如：
   - CLARITY
   - FOCUS
   - IMPACT

#### 验收标准

1. 文案简洁、有辨识度。
2. 不使用空泛口号，应与服务定位一致。

### 7.7 联系合作模块

#### 功能说明

作为页面主要转化入口，引导潜在客户发起联系。

#### 内容要求

1. 模块标题，例如：有一个值得解决的问题？
2. 联系方式：
   - 邮箱
   - 或表单入口
   - 或微信 / 企业微信

#### 本期建议

当前版本先使用邮箱链接，后续可扩展为表单。

#### 验收标准

1. 点击邮箱可唤起邮件客户端。
2. 联系方式为真实可用信息。
3. CTA 在页面中足够明显。

### 7.8 页脚

#### 功能说明

提供品牌收尾、版权信息和返回顶部入口。

#### 内容要求

1. 品牌名
2. 版权信息
3. 返回顶部链接

#### 验收标准

1. 版权年份正确。
2. 返回顶部链接可用。

## 8. 内容需求

正式上线前需要准备以下内容：

1. 公司正式名称
2. Logo 或品牌符号
3. 品牌一句话定位
4. 主标题和副标题
5. 3 个核心服务说明
6. 2-3 个代表案例
7. 案例结果数据
8. 联系邮箱 / 微信 / 表单地址
9. ICP 备案信息，如在中国大陆正式部署
10. 隐私政策，如接入表单、统计或广告追踪

## 9. 视觉与体验要求

### 9.1 视觉风格

整体风格为“现代科技 × 极简商务”。

### 9.2 色彩规范

1. 主色：深海蓝 `#07182D`
2. 强调色：电光蓝 `#3978FF`
3. 辅助强调：荧光绿 `#C9FF57`
4. 背景色：暖白 `#F5F2E9`
5. 文本色：接近白色或深蓝，保证对比度

### 9.3 字体规范

1. 中文：苹方 / 微软雅黑
2. 英文：Inter / Aptos

### 9.4 体验原则

1. 信息层级清晰。
2. 首屏价值表达明确。
3. 页面节奏要有留白。
4. CTA 不宜过多，避免分散转化。
5. 移动端阅读体验优先。

## 10. 响应式要求

### 10.1 桌面端

适配宽度：1280px 及以上。  
要求：左右分栏、卡片展示、视觉层级完整。

### 10.2 平板端

适配宽度：768px - 1279px。  
要求：模块可适当收缩，保持主内容可读。

### 10.3 移动端

适配宽度：375px - 767px。  
要求：

1. 不出现横向滚动。
2. CTA 按钮可点击区域充足。
3. 标题不溢出。
4. 案例卡片纵向排列。

## 11. SEO 要求

### 11.1 基础 SEO

1. 页面需要唯一 `title`。
2. 页面需要 `description`。
3. 页面只保留一个 H1。
4. 各模块标题使用合理 H2 / H3 结构。
5. 图片或视觉元素需要语义化替代描述。

### 11.2 建议关键词

根据实际公司定位选择，例如：

1. AI 产品咨询
2. 数字化战略
3. 产品设计
4. 增长咨询
5. AI 自动化解决方案

## 12. 数据埋点要求

本期如需统计转化效果，建议埋点：

1. 首页访问量
2. 点击“查看案例”
3. 点击“了解服务”
4. 点击“开始合作”
5. 点击邮箱 / 表单提交
6. 页面滚动深度

## 13. 非功能要求

### 13.1 性能

1. 首屏加载应尽量控制在 2 秒内。
2. 不依赖大型前端框架。
3. 图片和视觉资源应压缩后使用。

### 13.2 可维护性

1. 文案、颜色、案例数据应便于替换。
2. 页面结构清晰，方便后续扩展为多页面网站。
3. 样式文件应保持模块化命名。

### 13.3 兼容性

至少支持：

1. Chrome 最新版
2. Safari 最新版
3. Edge 最新版
4. iOS Safari
5. Android Chrome

### 13.4 合规

1. 若部署在中国大陆服务器，需补充 ICP 备案信息。
2. 若收集用户信息，需要隐私政策。
3. 若使用第三方统计，需要在隐私政策中说明。

## 14. 验收标准

### 14.1 内容验收

1. 品牌名、联系方式、案例数据已替换为真实内容。
2. 页面不存在明显占位文案。
3. 服务能力描述与公司实际业务一致。

### 14.2 功能验收

1. 所有导航锚点可正常跳转。
2. 所有 CTA 可正常点击。
3. 邮箱链接或表单入口可正常使用。
4. 返回顶部可正常跳转。

### 14.3 视觉验收

1. 桌面端布局完整。
2. 移动端无横向滚动。
3. 字体、颜色、按钮、卡片风格统一。
4. 页面没有明显遮挡、错位、溢出。

### 14.4 技术验收

1. HTML 结构语义清晰。
2. CSS 可维护。
3. 页面可独立部署。
4. 无控制台明显错误。
5. Lighthouse Performance、Accessibility、SEO 建议均达到 80 分以上。

## 16. 主要风险与待确认项

### 16.1 风险

1. 案例数据如果不真实，会影响品牌可信度。
2. 服务范围过宽，可能导致用户无法判断核心优势。
3. 仅使用邮箱转化，可能丢失部分低意向线索。
4. 未做 SEO 和内容运营时，自然流量有限。

### 16.2 待确认项

1. 公司正式名称是否使用 NOVA，还是替换为真实品牌名。
2. 官网主定位是否聚焦 AI 产品、数字战略、增长咨询，或其他方向。
3. 是否已有真实案例和客户数据。
4. 联系方式使用邮箱、微信、表单还是预约链接。
5. 是否需要部署上线，以及部署到哪个域名。
6. 是否需要备案、隐私政策、统计工具。

## 17. 一句话需求摘要

建设一个现代科技感、极简商务风格的公司官网，用清晰的首页结构展示品牌定位、服务能力、案例成果和合作入口，帮助潜在客户快速理解公司价值并发起咨询。
