## 1. 技术结论

V1.5 采用纯静态前端实现，不新增后端、不新增接口、不保存数据、不接入埋点、不外发问卷结果。

本期只改造：

1. `questionnaire.html`
2. `questionnaire.js`
3. `styles.css`
4. 必要时同步调整 `index.html#contact` 联系区文案

核心技术方案：

```text
HTML 静态结构
→ JavaScript 本地计算
→ Canvas 原生绘制雷达图
→ DOM 渲染总分、8 维度、Top 3
→ Clipboard API 复制摘要
→ 失败时展示手动复制文本区
→ CTA 跳转 index.html#contact
```

## 5. 计算规则

### 5.1 总分

```text
总分 = round(全部 24 题分值总和 / 120 × 100)
```

### 5.2 维度分

```text
维度分 = round(该维度 3 题分值总和 / 15 × 100)
```

### 5.3 总分等级

| 总分 | 等级 | CMMI 参考映射 | 建议 |
|---:|---|---|---|
| 85-100 | 高成功概率 | 参考映射：CMMI L4-L5 | 强化数据驱动和组织级最佳实践沉淀 |
| 70-84 | 可控推进 | 参考映射：CMMI L3-L4 | 加强风险前置、发布门禁和复盘闭环 |
| 55-69 | 需要治理 | 参考映射：CMMI L2-L3 | 优先补齐计划、依赖、风险和质量机制 |
| 40-54 | 高风险 | 参考映射：CMMI L1-L2 | 先做项目治理重构再加速交付 |
| 0-39 | 极高风险 | 参考映射：CMMI L1 | 当前更适合先做基础治理梳理，优先完成目标、范围、组织和质量体系梳理 |

### 5.4 维度状态

| 维度分 | 状态 | class |
|---:|---|---|
| 85-100 | 优势维度 | `is-strong` |
| 70-84 | 稳定维度 | `is-stable` |
| 55-69 | 待加强 | `is-warning` |
| 40-54 | 风险维度 | `is-risk` |
| 0-39 | 核心短板 | `is-critical` |

### 5.5 Top 3 排序

排序规则：

1. 按维度分从低到高。
2. 分数相同时按 A-H 原始顺序。
3. 取前 3 个。

## 7. JavaScript 模块设计

在现有 `questionnaire.js` 内组织为以下函数。

| 函数 | 责任 |
|---|---|
| `readAnswers()` | 读取 24 个 select 的当前值 |
| `calculateResult(answers)` | 计算总分、等级、维度分、Top 3 |
| `getScoreLevel(totalScore)` | 返回等级、CMMI 参考映射和总体建议 |
| `getDimensionStatus(score)` | 返回维度状态和 class |
| `renderScore(result)` | 渲染总分、等级、建议 |
| `renderDimensions(result.dimensions)` | 渲染 8 维列表 |
| `renderWeaknesses(result.weaknesses)` | 渲染 Top 3 |
| `drawRadar(result.dimensions)` | Canvas 绘制雷达图 |
| `generateSummary(result)` | 生成摘要文本 |
| `copySummary()` | 调用 Clipboard API，失败时展示手动复制 |
| `handleResize()` | 防抖后重绘雷达图 |

## 8. Canvas 雷达图方案

### 8.1 绘制规则

1. 使用 8 个轴。
2. 使用 4-5 层网格。
3. 标签使用短词：目标、组织、需求、计划、质量、数据、工具、复盘。
4. 初始 0 分时绘制中心点和说明。
5. 维度分变化后重绘。

### 8.2 高清屏适配

实现原则：

```js
const ratio = window.devicePixelRatio || 1;
canvas.width = cssWidth * ratio;
canvas.height = cssHeight * ratio;
ctx.setTransform(ratio, 0, 0, ratio, 0, 0);
```

### 8.3 resize

1. 监听 `window.resize`。
2. 使用 100-200ms 防抖。
3. resize 后重新读取 canvas 容器尺寸并重绘。

## 9. 复制摘要方案

### 9.1 成功路径

```text
点击复制
→ navigator.clipboard.writeText(summaryText)
→ 成功
→ 按钮显示“已复制”
→ 1.5-2 秒后恢复“复制诊断摘要”
```

### 9.2 失败路径

失败场景：

1. Clipboard API 不可用。
2. 用户拒绝剪贴板权限。
3. 浏览器安全策略限制。
4. 其他异常。

降级：

```text
展示 manualSummary
→ 填入 summaryText
→ 用户手动复制
```

失败时不上传、不提交、不发送任何数据。

## 11. 数据与隐私控制

技术硬约束：

1. 不新增接口请求。
2. 不新增 `fetch` / `XMLHttpRequest`。
3. 不新增统计脚本。
4. 不新增表单提交。
5. 不使用 localStorage / sessionStorage 保存问卷结果。
6. 不自动发送邮件、飞书或第三方请求。
7. 复制摘要只写入本地剪贴板。

检查方式：

1. 代码搜索 `fetch`、`XMLHttpRequest`、`sendBeacon`、`localStorage`、`sessionStorage`。
2. 浏览器 Network 面板验证问卷交互无新增请求。

## 13. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| Canvas 在小屏标签拥挤 | 用户看不清 | 使用短标签，维度列表兜底 |
| Clipboard API 被限制 | 用户无法复制 | 展示手动复制文本框 |
| 计算结果和摘要不一致 | 咨询判断错误 | 页面和摘要共用同一计算结果对象 |
| 开发误加接口或埋点 | 违反 V1.5 数据策略 | 代码搜索 + Network 检查 |
| 左侧结果栏过长 | 首屏拥挤 | 8 维列表放右侧或下方 |
