# 北京润序科技官网 V1.5 诊断结果增强设计方案 PRD 对齐版

版本：V1.5 PRD 对齐版  
日期：2026-07-14  
设计角色：UI/UX 设计师 / B2B 咨询转化设计师  
对应 PRD：`北京润序科技官网V1.5诊断结果PRD.md`  
关联页面：`OPC/网站/runxu-tech/questionnaire.html`

## 1. 设计结论

本设计方案严格对齐 V1.5 修订版 PRD。

V1.5 不重做整站，不建设后台，不增加表单提交，不自动发送飞书或邮件。设计目标是把问卷页从“分数展示页”升级为“前端本地诊断结果页”：

```text
总分
→ CMMI 参考映射
→ 8 维度雷达图
→ 8 维度得分列表
→ Top 3 短板
→ 复制诊断摘要
→ 跳转联系区
→ 用户主动通过飞书或邮箱发送摘要
```

核心原则：

1. 结果可读：用户能看懂总分、维度分和短板。
2. 交接可用：项目经理能根据摘要判断诊断重点。
3. 转化明确：先复制诊断摘要，再预约项目诊断。
4. 合规清晰：CMMI 只表达为“参考映射”，不是官方评估或认证。
5. 数据克制：V1.5 不上传、不保存、不埋点、不外发数据。

## 2. 页面信息架构

### 2.1 桌面端结构

```text
顶部导航
└─ 项目成功率自评说明

主体区
├─ 左侧诊断结果栏
│  ├─ 项目成功率得分
│  ├─ 风险等级 + CMMI 参考映射
│  ├─ 建设性总体建议
│  ├─ 8 维度雷达图
│  ├─ 雷达图 0 分说明
│  ├─ 优先治理 Top 3
│  ├─ 复制诊断摘要
│  ├─ 生成诊断摘要并预约
│  └─ V1.5 数据说明
│
└─ 右侧问卷与明细区
   ├─ A-H 8 组问卷
   ├─ 8 维度得分列表
   └─ 手动复制摘要区
```

设计取舍：

- 左侧只放“用户决策和转化需要的信息”，避免把 8 维完整列表塞进左栏导致拥挤。
- 右侧承载问卷填写和完整维度分数，保证结果准确可查。
- 复制失败的手动文本区放在右侧明细区或按钮下方，不遮挡问卷。

### 2.2 移动端结构

严格按 PRD 顺序：

```text
顶部导航
问卷说明
总分与风险等级
雷达图
Top 3 短板
复制摘要 / 预约诊断
问卷题目
8 维度得分列表
手动复制文本区
```

移动端要求：

1. 375px 不横向滚动。
2. 下拉框宽度不超过屏幕。
3. 雷达图控制在 280-320px。
4. 主 CTA 可简化为 `预约项目诊断`，但按钮旁必须保留“建议先复制诊断摘要”的说明。

## 3. 视觉规范

### 3.1 风格定位

深海蓝科技感 × 项目诊断仪表盘 × 高端咨询服务。

页面不能像普通问卷，也不能像后台系统。它应该像一个轻量的“项目治理诊断仪表盘”，既有专业感，也能促成用户联系。

| 深色卡片 | `#0B1424` / `#111C2E` | 诊断结果栏 |

### 3.3 文案口径

必须使用：

- `参考映射：CMMI L1`
- `本工具用于项目管理诊断沟通，不等同于官方 CMMI 评估或认证。`
- `V1.5 不上传数据，请复制摘要后通过飞书或邮箱发送。`
- `建议先复制诊断摘要，再预约诊断。`

不得使用：

- `CMMI 认证结果`
- `CMMI 官方评估`
- `系统已提交结果`
- `后台已收到数据`

## 4. 核心模块设计

### 4.1 总分与风险等级

位置：左侧诊断结果栏顶部。

展示内容：

- 标题：`项目成功率得分`
- 大数字：`0-100`
- 等级：如 `极高风险 · 参考映射：CMMI L1`
- 建议：建设性短句，不只输出风险结论。

初始状态：

```text
0
极高风险 · 参考映射：CMMI L1
当前更适合先做基础治理梳理，建议优先明确目标、范围、组织职责和质量机制。
```

设计要求：

1. 总分使用荧光绿，形成第一视觉焦点。
2. 风险等级紧贴总分，不要让用户只看到数字。
3. 风险文案必须带行动建议，避免“吓退式诊断”。
4. 默认 0 分是有效状态，不是空状态。

### 4.2 雷达图

位置：总分与风险等级下方。

展示内容：

- 8 个短标签：目标、组织、需求、计划、质量、数据、工具、复盘。
- 分值范围：0-100。
- 初始状态：全部 0 分。
- 0 分状态也必须显示网格、标签、中心点和说明。

0 分说明：

```text
当前各维度均为 0 分
```

设计要求：

1. 雷达图用于判断强弱趋势，不承担精确读数。
2. 具体数值由 8 维度列表展示。
3. 雷达填充使用半透明电光蓝。
4. 坐标网格使用低透明浅蓝。
5. Canvas 需适配 `devicePixelRatio`，窗口 resize 时重绘。

### 4.3 Top 3 短板

位置：左侧诊断结果栏，雷达图下方。

展示内容：

- 标题：`优先治理 Top 3`
- 排名
- 维度名称
- 维度得分
- 对应治理建议

初始状态：

```text
1. 目标与范围 0：优先明确业务目标、用户目标、成功指标、本期范围和不做事项。
2. 组织与职责 0：建立项目 Owner、RACI 职责矩阵、跨团队依赖响应机制。
3. 需求与变更 0：建立 PRD/原型/技术方案准入，明确变更评估、审批和排期影响。
```

设计要求：

1. Top 3 是项目经理最需要的信息，展示优先级高于完整 8 维列表。
2. 每条建议控制在 1-2 行。
3. 同分时按 A-H 顺序展示。
4. A/B/C 提高后，Top 3 应自动顺延到当前最低维度。

### 4.4 复制诊断摘要 / 预约诊断

位置：左侧诊断结果栏底部。

按钮层级：

| 按钮 | 层级 | 文案 |
|---|---|---|
| 复制摘要 | 次级按钮 | `复制诊断摘要` |
| 预约诊断 | 主按钮 | `生成诊断摘要并预约` |

按钮说明：

```text
建议先复制诊断摘要，再通过飞书或邮箱发送给我们。
V1.5 不上传数据，请复制摘要后通过飞书或邮箱发送。
```

交互状态：

1. 点击复制后，成功写入剪贴板。
2. 成功后按钮显示 `已复制`，保持 1.5-2 秒后恢复。
3. 复制失败时展示手动复制文本区。
4. 点击主 CTA 跳转：`index.html#contact`。
5. 用户未复制摘要也允许跳转，但页面必须提示建议先复制。

### 4.5 8 维度得分列表

位置：桌面端右侧问卷下方；移动端问卷下方。

每个维度展示：

- 维度名称
- 分数
- 状态
- 进度条

状态规则：

| 分数 | 状态 | 视觉 |
|---:|---|---|
| 85-100 | 优势维度 | 绿色 |
| 70-84 | 稳定维度 | 蓝色 |
| 55-69 | 待加强 | 黄绿 |
| 40-54 | 风险维度 | 橙色 |
| 0-39 | 核心短板 | 红橙 |

设计要求：

1. 不放入左侧结果栏，避免首屏拥挤。
2. 分数必须明确可读。
3. 雷达图和维度列表共用同一套计算结果。

### 4.6 手动复制摘要区

触发条件：

- Clipboard API 不可用。
- 浏览器权限拒绝。
- 非 HTTPS 或本地环境导致剪贴板写入失败。
- 其他复制异常。

展示内容：

- 标题：`请手动复制以下诊断摘要`
- 文本框：完整摘要
- 说明：`当前浏览器未开放剪贴板权限，内容未上传。`

设计要求：

1. 默认隐藏。
2. 展开后至少显示 6-8 行。
3. 不能遮挡主 CTA。
4. 文本框方便用户全选复制。

## 5. 诊断摘要设计

摘要必须短、结构化、可直接转发。

```text
【Runxu Tech 项目成功率自评摘要】
总分：42
等级：高风险 · 参考映射：CMMI L1-L2

8 维度得分：
- 目标与范围：40
- 组织与职责：53
- 需求与变更：33
- 计划与风险：47
- 质量与测试：47
- 数据与效能：38
- 工具链与流程：52
- 复盘与持续改进：44

短板 Top 3：
1. 需求与变更 33：建立 PRD/原型/技术方案准入，明确变更评估、审批和排期影响。
2. 数据与效能 38：建立进度、质量、效率、风险指标和可视化效能看板。
3. 目标与范围 40：优先明确业务目标、用户目标、成功指标、本期范围和不做事项。

说明：本工具用于项目管理诊断沟通，不等同于官方 CMMI 评估或认证，不是完整解决方案。
```

摘要不包含：

1. 24 道题逐题答案。
2. 用户身份信息。
3. 联系方式。
4. 系统生成的用户 ID。

## 6. 联系区承接设计

联系区必须承接问卷页转化，不只展示二维码和邮箱。

新增提示文案：

```text
请将诊断摘要通过飞书或邮箱发送给我们，便于快速判断项目治理重点。
```

联系区动作：

- 飞书二维码：用户扫码添加。
- 邮箱：`Laura_fan@126.com`
- 用户主动发送摘要。

不得表达：

- 系统已收到问卷。
- 后台已保存结果。
- 自动发送给项目经理。

## 7. 数据与隐私设计

页面需要在按钮区或结果区明确说明：

```text
V1.5 不上传数据，请复制摘要后通过飞书或邮箱发送。
```

设计边界：

1. 不新增接口。
2. 不新增表单提交。
3. 不接入统计脚本。
4. 不发送埋点。
5. 不自动发送飞书或邮件。
6. 不保存问卷结果。
7. 页面刷新后恢复默认 0 分。

## 8. 开发交付约束

建议元素 ID：

| 元素 | 建议 ID |
|---|---|
| 雷达图 | `radarChart` |
| 8 维度列表 | `dimensionList` |
| Top 3 短板 | `weaknessList` |
| 复制按钮 | `copySummary` |
| 手动复制文本框 | `manualSummary` |
| 预约 CTA | `diagnosticContact` |

建议状态 class：

| 状态 | class |
|---|---|
| 优势维度 | `is-strong` |
| 稳定维度 | `is-stable` |
| 待加强 | `is-warning` |
| 风险维度 | `is-risk` |
| 核心短板 | `is-critical` |

数据要求：

1. 页面显示、雷达图、Top 3、摘要必须共用同一套计算结果。
2. Canvas 初始化、分值变化、窗口 resize 时重绘。
3. 不引入第三方图表库。
4. 24 个问题变化后，页面不得出现明显卡顿。
