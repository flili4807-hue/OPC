# 北京润序科技官网 V2.0 获客闭环 UI/UX 设计方案

版本：V2.0 修订版  
日期：2026-07-14  
设计角色：UI/UX 设计师 / B2B 获客转化设计师  
对应 PRD：`北京润序科技官网V2.0获客闭环PRD.md`  
修订依据：`北京润序科技官网V2.0获客闭环设计方案_多角色评审记录.md`  
P1 修订依据：`北京润序科技官网V2.0获客闭环PRD与设计方案_联合评审记录.md`  
关联页面：`questionnaire.html`、`index.html#contact`

## 1. 设计结论

V2.0 的设计目标不是重做问卷页视觉，而是在 V1.5 的诊断结果基础上增加“留资换项目管理自评诊断报告”的获客闭环。

核心设计策略：

```text
先给用户看到基础诊断价值
→ 再引导用户提交咨询信息
→ 解锁项目管理自评诊断报告
→ 自动沉淀线索并触发邮件 / 飞书通知
```

设计重点：

1. 留资不能太早，否则用户没有动力提交。
2. 报告不能完全免费开放，否则无法沉淀客户。
3. 表单不能太重，否则转化率下降。
4. 授权和隐私说明必须明确，避免用户不信任。
5. 解锁后的报告要让用户觉得“值得提交信息”。

本次 P1 修订重点：

1. 明确解锁前只展示基础结果，解锁后展示完整治理建议。
2. 强化项目管理自评诊断报告的价值表达，避免用户不知道为什么要留资。
3. 重构留资弹窗信息层级，降低填写阻力。
4. 移动端统一使用整页卡片，保证按钮和输入区可达。
5. 增加前端状态命名，便于研发和测试对齐。
6. 关闭表单时返回问卷结果页，并保留本次已填写内容。

## 2. 用户体验目标

### 2.1 用户侧

用户需要感受到：

1. 我已经完成了一次有价值的诊断。
2. 当前页面已经给了我基本结果。
3. 项目管理自评诊断报告需要提交少量咨询信息。
4. 提交后能马上看到更完整的建议。
5. 我的信息只用于项目诊断沟通。

### 2.2 业务侧

官网需要实现：

1. 把问卷完成用户转化为可跟进线索。
2. 获取姓名、公司、联系方式、项目背景。
3. 把诊断结果和客户信息一起发送给售前 / 创始人。
4. 降低用户跳出和无效咨询。

### 3.1 解锁前结构

```text
顶部导航
问卷说明

主体区
├─ 左侧诊断结果栏
│  ├─ 总分
│  ├─ 风险等级
│  ├─ CMMI 参考映射
│  ├─ 雷达图
│  ├─ Top 3 短板名称 + 分数
│  ├─ 自评诊断报告价值提示
│  ├─ 生成项目管理自评诊断报告 CTA
│  └─ 复制当前摘要
│
└─ 右侧问卷区
   ├─ A-H 问题
   └─ 8 维度得分列表
```

解锁前可见：

- 总分
- 风险等级
- CMMI 参考映射
- 雷达图
- 8 维度分数
- Top 3 短板名称 + 分数
- 自评诊断报告价值提示

解锁前隐藏：

- Top 3 完整治理建议
- 完整报告摘要
- 下一步诊断建议
- 已提交成功提示

解锁前 Top 3 展示规则：

```text
展示：维度名称 + 维度分数
隐藏：完整治理建议、行动建议、报告摘要
提示：提交咨询信息后，查看 Top 3 治理建议，并带着报告预约项目诊断
```

### 3.2 留资表单结构

桌面端：居中弹窗。  
移动端：整页卡片。

```text
弹窗标题：生成项目管理自评诊断报告
价值说明：提交咨询信息后，立即查看完整治理建议、雷达图解读和下一步诊断重点

字段：
姓名 *
公司 / 团队 *
联系方式 *
职位 / 角色
项目背景 *
授权勾选 *

隐私入口：
查看信息使用说明

按钮：
提交并生成报告
取消
```

弹窗信息层级：

1. 标题区：生成项目管理自评诊断报告。
2. 价值说明区：说明提交后可查看完整治理建议、雷达图解读和下一步诊断重点。
3. 字段区：按“姓名 → 公司 → 联系方式 → 职位 → 项目背景”排列。
4. 授权区：明确用户同意将信息用于项目诊断沟通。
5. 隐私入口：放在授权区附近，支持打开说明弹层。
6. 按钮区：主按钮“提交并生成报告”，次按钮“取消”。

关闭弹窗规则：

- 点击取消、关闭图标或遮罩后，返回当前问卷结果页。
- 不解锁项目管理自评诊断报告。
- 本次页面会话内保留已填写内容，用户再次打开表单时自动恢复。
- 已填写内容只保留在前端页面状态中，未提交前不进入后台。
- 本期采用页面会话解锁：线索保存成功后在当前页面解锁报告，刷新页面后回到未解锁状态。

### 3.3 解锁后结构

```text
提交成功提示
项目管理自评诊断报告
├─ 自评诊断报告价值说明
├─ 总分 / 等级 / CMMI 参考映射
├─ 雷达图
├─ 8 维度得分
├─ Top 3 短板
├─ Top 3 治理建议
├─ 下一步诊断建议
├─ 免责声明
└─ 操作按钮
   ├─ 复制报告摘要
   └─ 返回首页
```

解锁后价值说明：

```text
项目管理自评诊断报告已解锁，包含 8 维度得分、Top 3 治理建议和下一步诊断重点。项目经理可据此快速判断优先治理方向，并带着结果预约首次项目诊断。
```

解锁条件说明：

- 服务端校验通过且线索保存成功后，进入 `unlocked`。
- 邮件或飞书通知失败不影响报告解锁，但失败状态由后端记录。
- 线索保存失败时保持未解锁，并展示重试、复制摘要和邮箱联系入口。

## 4. 核心交互设计

### 4.1 主 CTA

V2.0 主 CTA：

```text
生成项目管理自评诊断报告
```

辅助说明：

```text
提交咨询信息后，立即查看完整治理建议和下一步诊断重点。润序科技会基于评测结果与你联系。
```

按钮层级：

| 按钮 | 层级 | 使用位置 |
|---|---|---|
| 生成项目管理自评诊断报告 | 主按钮 | 结果栏底部 |
| 复制当前摘要 | 次级按钮 | 主按钮下方 |
| 提交并生成报告 | 主按钮 | 留资表单 |
| 取消 | 文本按钮 | 留资表单 |

### 4.2 留资表单

字段排序：

1. 姓名
2. 公司 / 团队
3. 联系方式
4. 职位 / 角色
5. 项目背景
6. 授权勾选

排序理由：

- 先收最容易填写的信息，降低压力。
- 联系方式提前，保证核心线索字段优先完成。
- 项目背景放后面，适合作为长文本收尾。
- 授权放提交前，符合用户决策顺序。

表单文案：

```text
我们会把你的评测结果和项目信息发送给润序科技，用于生成项目管理自评诊断报告、后续项目诊断沟通和方案建议。
```

项目背景占位文案：

```text
请简单描述项目类型、当前阶段、主要问题或你希望优先诊断的方向。
```

移动端表单规则：

1. 使用整页卡片，不使用全局悬浮弹窗。
2. 字段单列纵向排列。
3. 项目背景放在最后一个输入字段，textarea 高度不低于 120px。
4. 授权区在提交按钮上方，文案必须完整可读。
5. 提交按钮固定在表单内容底部，不遮挡输入框。
6. 键盘弹起时，当前输入框和错误提示必须可见。

### 4.3 隐私说明弹层

入口文案：

```text
查看信息使用说明
```

弹层内容：

```text
我们将收集你主动提交的姓名、公司 / 团队、职位 / 角色、联系方式、项目背景、本次问卷答案和诊断结果。

这些信息仅用于北京润序科技有限责任公司进行项目诊断沟通、售前跟进和服务改进。

信息默认保存 180 天。你可以通过 Laura_fan@126.com 联系我们删除相关信息。

本工具用于项目管理诊断沟通，不等同于官方 CMMI 评估或认证。
```

### 4.4 提交中状态

点击提交后：

- 按钮文案：`提交中...`
- 按钮禁用
- 表单字段不可重复提交
- 保留页面，不跳转

### 4.5 提交成功状态

成功提示：

```text
诊断报告已生成，我们已收到你的咨询信息。润序科技将基于评测结果与你联系，通常会在 1-2 个工作日内响应。
```

视觉表现：

- 成功图标
- 绿色或蓝色成功提示条
- 报告区域展开
- CTA 从“生成项目管理自评诊断报告”变为“复制报告摘要”

### 4.6 提交失败状态

失败提示：

```text
提交失败，请稍后重试。你也可以复制诊断摘要，通过飞书或邮箱联系我们。
```

按钮：

- `重试提交`
- `复制诊断摘要`
- `通过邮箱联系`

设计要求：

1. 不丢失用户已填写表单内容。
2. 不隐藏当前诊断结果。
3. 不直接暴露邮件或飞书内部错误详情。

### 4.7 前端状态命名

| 状态 | 含义 | 页面表现 |
|---|---|---|
| `locked` | 未提交咨询信息，项目管理自评诊断报告未解锁 | 展示基础结果、雷达图、Top 3 名称和分数 |
| `form_open` | 用户打开留资表单 | 展示表单，保留当前问卷结果 |
| `submitting` | 表单提交中 | 禁用按钮和字段，显示提交中 |
| `unlocked` | 线索保存成功，报告已解锁 | 展示项目管理自评诊断报告、治理建议和下一步诊断重点 |
| `failed` | 提交失败 | 展示失败提示，允许重试、复制摘要和邮箱联系 |

状态流转：

```text
locked
→ 点击生成项目管理自评诊断报告
→ form_open
→ 提交
→ submitting
→ 线索保存成功 unlocked
→ 失败 failed
```

关闭表单流转：

```text
form_open → 取消 / 关闭 → locked
```

关闭后仍保留本次已填写内容，再次进入 `form_open` 时恢复。

## 5. 项目管理自评诊断报告设计

### 5.1 报告头部

展示：

- 项目成功率自评得分
- 风险等级
- CMMI 参考映射
- 生成时间

示例：

```text
项目成功率自评得分：42
高风险 · 参考映射：CMMI L1-L2
生成时间：2026-07-14 18:30
```

### 5.2 雷达图与 8 维列表

沿用 V1.5 设计。

要求：

1. 雷达图显示维度强弱趋势。
2. 8 维列表显示具体分数和状态。
3. 分数和状态保持一致。

### 5.3 Top 3 治理建议

解锁后展示完整建议。

解锁前后展示差异：

| 内容 | 解锁前 | 解锁后 |
|---|---|---|
| Top 3 维度名称 | 展示 | 展示 |
| Top 3 维度分数 | 展示 | 展示 |
| 风险判断 | 隐藏 | 展示 |
| 治理建议 | 隐藏 | 展示 |
| 下一步诊断建议 | 隐藏 | 展示 |
| 完整报告摘要 | 隐藏 | 展示并支持复制 |

每条包含：

- 维度名称
- 分数
- 风险判断
- 治理建议

示例：

```text
需求与变更 33 分
建议建立 PRD / 原型 / 技术方案准入，明确变更评估、审批和排期影响。
```

### 5.4 下一步诊断建议

根据 Top 3 生成一句行动建议。

示例：

```text
建议首次诊断优先确认需求变更机制、质量门禁和跨团队依赖响应机制。
```

### 5.5 报告免责声明

固定展示：

```text
本工具用于项目管理诊断沟通，不等同于官方 CMMI 评估或认证，不是完整解决方案。
```

## 6. 响应式设计

### 6.1 桌面端

宽度 ≥ 1024px：

- 左侧结果栏 sticky
- 右侧问卷和报告内容滚动
- 留资表单居中弹窗
- 弹窗宽度建议 560-640px

### 6.2 平板端

768-1023px：

- 单列或弱双列
- 结果区在上
- 问卷区在下
- 留资表单可居中弹窗或整页卡片，优先保证输入可达

### 6.3 移动端

≤ 767px：

- 单列布局
- 结果区在前
- CTA 固定在结果区底部，不做全局悬浮
- 留资表单使用整页卡片
- 项目背景 textarea 高度不低于 120px
- 授权文案不能过小
- 提交按钮必须在 375px 宽度下可直接点击
- 键盘弹起后，当前输入框、错误提示和提交按钮不能被遮挡
- 关闭入口固定在表单卡片右上角

## 7. 表单校验设计

### 7.1 校验方式

采用字段下方即时提示 + 提交时统一校验。

错误文案：

| 场景 | 文案 |
|---|---|
| 姓名为空 | 请填写姓名 |
| 公司为空 | 请填写公司或团队名称 |
| 联系方式为空 | 请填写手机号、邮箱或飞书号 |
| 联系方式无效 | 请填写有效联系方式 |
| 项目背景过短 | 请简单说明项目背景或当前困扰 |
| 未授权 | 请确认授权后生成诊断报告 |

### 7.2 错误视觉

- 字段边框变为风险橙或红色
- 错误文案显示在字段下方
- 首个错误字段自动聚焦
- 不使用浏览器默认 alert

状态样式建议：

| 状态类名 | 使用场景 | 视觉表现 |
|---|---|---|
| `is-invalid` | 字段校验失败 | 边框高亮，字段下方显示错误文案 |
| `is-submitting` | 表单提交中 | 按钮禁用，显示提交中 |
| `is-unlocked` | 报告已解锁 | 项目管理自评诊断报告区域展开 |
| `is-failed` | 提交失败 | 失败提示条出现，保留表单内容 |

## 8. 内容与品牌要求

### 8.1 文案语气

专业、克制、可信。

避免：

- 夸大承诺
- “官方评估”
- “认证结果”
- “系统已生成正式报告”

推荐表达：

- “项目管理诊断沟通”
- “CMMI 参考映射”
- “初步治理建议”
- “1-2 个工作日内响应”

### 8.2 信息安全感

在表单底部明确说明：

```text
我们仅将你提交的信息用于本次项目诊断沟通，不会公开展示你的联系方式或评测结果。
```

## 9. 设计验收标准

### 9.1 主流程

- [ ] 点击“生成项目管理自评诊断报告”后展示留资表单。
- [ ] 桌面端为居中弹窗。
- [ ] 移动端为整页卡片。
- [ ] 留资表单符合“标题 → 价值说明 → 字段 → 授权 → 隐私入口 → 按钮”的层级。
- [ ] 表单字段顺序符合设计方案。
- [ ] 必填字段未填时不能提交。
- [ ] 授权未勾选时不能提交。
- [ ] 提交中按钮禁用。
- [ ] 线索保存成功后展示项目管理自评诊断报告。
- [ ] 解锁后展示完整治理建议。
- [ ] 关闭表单后返回问卷结果页，并保留本次已填写内容。

### 9.2 解锁控制

- [ ] 未留资前不展示完整治理建议。
- [ ] 未留资前不展示完整报告摘要。
- [ ] 未留资前 Top 3 只展示维度名称和分数。
- [ ] 留资且线索保存成功后展示项目管理自评诊断报告。
- [ ] 留资失败时不丢失用户填写内容。
- [ ] `locked`、`form_open`、`submitting`、`unlocked`、`failed` 状态表现可区分。
- [ ] 邮件或飞书通知失败但线索保存成功时，前端仍展示已解锁报告。
- [ ] 刷新页面后回到未解锁状态。

### 9.3 合规与信任

- [ ] 授权文案清晰可读。
- [ ] 隐私说明弹层可打开。
- [ ] 隐私说明包含收集范围、用途、保存周期和删除方式。
- [ ] 所有 CMMI 表达均为“参考映射”。
- [ ] 页面不出现“官方认证”或“认证结果”。

### 9.4 响应式

- [ ] 375px 下无横向滚动。
- [ ] 375px 下项目背景输入框、授权文案和提交按钮可正常操作。
- [ ] 768px 下表单可正常填写。
- [ ] 1440px 下弹窗宽度和结果栏比例稳定。

## 10. 设计风险与应对

| 风险 | 影响 | 设计应对 |
|---|---|---|
| 留资太早导致流失 | 用户不提交 | 先展示基础诊断结果，再引导解锁项目管理自评诊断报告 |
| 表单字段太多 | 转化下降 | 字段控制在 5 个，职位选填 |
| 授权文案过长 | 用户忽略或不信任 | 简短授权 + 可展开隐私说明 |
| 移动端弹窗难填写 | 输入体验差 | 移动端使用整页卡片 |
| 报告价值感不足 | 用户觉得不值得留资 | 解锁后展示完整治理建议和下一步诊断建议 |
