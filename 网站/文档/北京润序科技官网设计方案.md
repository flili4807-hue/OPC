# 北京润序科技官网设计方案

版本：V1.5  
日期：2026-07-14  
设计角色：官网产品设计师 / 品牌视觉设计师  
对应 PRD：`北京润序科技官网PRD.md`  
当前页面：`OPC/网站/runxu-tech/index.html`、`OPC/网站/runxu-tech/questionnaire.html`

## 1. 设计目标

本官网设计目标不是做“传统企业介绍页”，而是建立一个专家背书驱动的 B2B 咨询服务官网。

页面需要在短时间内完成三件事：

1. 让访问者理解北京润序科技是谁。
2. 让访问者相信公司具备复杂项目交付和 PMO 体系建设能力。
3. 引导访问者预约 30 分钟项目诊断，通过邮箱或飞书二维码发起咨询。

V1.2 的核心修订目标：

1. 将“预约咨询”升级为更具体的“预约 30 分钟项目诊断”。
2. 增加典型痛点和服务包，减少“能力展示”感，强化“可购买服务”。
3. 补齐问卷未完成、完成后、咨询引导等状态设计。
4. 增加案例公开策略、SEO 分享信息和可测试验收标准。

V1.3 的核心修订目标：

1. 重新设计项目成功率自评问卷页，修复白底低对比问题。
2. 问卷页采用深海蓝诊断仪表盘风格，与官网主视觉统一。
3. 增加问卷摘要卡：24 个问题、8 个治理维度、15 分钟预计完成。
4. 优化评分侧栏和问题卡片对比度，确保标题、题目和选择器可读。

V1.5 的核心修订目标：

1. 问卷所有题目下拉框默认值改为 `0 分`，表示尚未具备或没有选择有效能力。
2. 页面打开即展示默认诊断结果：0 分，极高风险 · CMMI L1。
3. 用户可在 0-5 分之间调整，系统实时更新评分和建议。

## 2. 设计关键词

| 关键词 | 设计含义 |
|---|---|
| 专业 | 信息层级克制、文案直接、避免营销感过重 |
| 可信 | 强化真实经历、数据结果、认证与方法论 |
| 科技 | 深海蓝全站底色、电光蓝强调、数据图形、仪表盘感 |
| 秩序 | 网格系统、序号、流程化模块、清晰分割线 |
| 高端服务 | 深色留白、大标题、玻璃拟态卡片、强 CTA |

## 3. 视觉方向

### 3.1 风格定位

深海科技 × 极简商务 × 专家咨询。

视觉上应接近“AI 咨询公司 / 技术管理顾问 / 高端 PMO 服务”的表达，而不是个人简历页或传统软件公司官网。

V1.1 统一采用深海蓝科技底色，叠加轻量网格、径向光晕和玻璃拟态卡片。原则是“整站一个空间感”，不再通过白、蓝、灰大块背景切换制造层级，改为用卡片透明度、边框、标题和电光蓝强调建立层次。

### 3.2 色彩规范

| 用途 | 色值 | 使用场景 |
|---|---|---|
| 深海蓝 | `#06101F` | 页面主背景、整体科技空间 |
| 暗蓝渐变 | `#07172B` / `#050B16` | 页面纵深、Hero 和 CTA 容器 |
| 电光蓝 | `#3978FF` | CTA、关键强调、品牌识别 |
| 荧光绿 | `#C9FF57` | 少量数据强调 |
| 冷白 | `#F5F8FF` | 主标题和正文主色 |
| 玻璃白 | `rgba(255,255,255,.075)` | 卡片背景 |
| 灰蓝 | `#95A8C0` | 辅助文字 |
| 透明边框 | `rgba(255,255,255,.13)` | 卡片、分割线、导航边界 |

### 3.3 字体规范

中文：苹方 / 微软雅黑 / 系统无衬线  
英文：Inter / Aptos / 系统无衬线

标题应使用大字号、紧凑字距，形成专家型品牌的稳定感。正文保持 16-18px，行高 1.7-1.9，保证深色背景下的阅读舒适度。

## 4. 页面信息架构

```text
顶部导航
├─ Hero 首屏：定位 + 价值 + CTA
├─ 能力标签：AI Agent / 战略 PMO / 效能体系 / 复杂交付
├─ 数据背书：18年 / 90+ / 1000+ / 亿级 / 70%+ / 20%+
├─ 典型痛点：延期 / 协同低效 / 需求反复 / 质量不稳
├─ 服务能力：4个服务方向
├─ 服务包：项目诊断 / PMO体系搭建 / Agent交付陪跑 / 效能看板建设
├─ 代表案例：4个高可信案例
├─ 方法论：5步交付治理路径
├─ 诊断评测：CMMI × 项目成功率模型
├─ 自评问卷页：24题项目成功率评测
├─ 创始人背书：经历、认证、奖项
└─ 联系合作：邮箱 + 飞书二维码
```

信息顺序的设计逻辑：

1. 先说“我是谁、解决什么问题”。
2. 再用数据证明可信。
3. 接着说明能提供什么服务。
4. 再用案例证明做过。
5. 然后展示方法论和诊断模型，体现专业产品化能力。
6. 最后给出明确联系入口。

## 5. 首页模块设计

### 5.1 顶部导航

设计目标：轻量、稳定、商务感。

布局：

- 左侧：`RUNXU TECH` 品牌字标 + 抽象品牌符号。
- 中间：服务、案例、方法、评测、关于。
- 右侧：项目诊断 CTA。

交互：

- 点击导航滚动到对应模块。
- CTA 始终作为最重要转化按钮，桌面端文案为“项目诊断”或“预约 30 分钟项目诊断”，移动端优先使用短文案“项目诊断”。
- 移动端隐藏中间导航，只保留品牌与 CTA。

### 5.2 Hero 首屏

设计目标：30 秒内建立定位。

首屏内容：

- 小标签：`AI AGENT · STRATEGIC PMO · DELIVERY GOVERNANCE`
- 主标题：`AI Agent × 战略 PMO × 复杂项目交付`
- 副标题：说明“从 0 到 1 搭建可复制、可度量、可交付的项目管理体系”
- CTA：预约 30 分钟项目诊断 / 查看案例
- 右侧视觉：抽象效能看板，突出 `20%+` 产研效能提升

视觉要点：

- 左文右图，强化 B2B 官网首屏专业感。
- 主标题使用大字号，突出“AI Agent / 战略 PMO / 复杂项目交付”三大关键词。
- 右侧不使用真实截图，避免泄露客户信息，用抽象数据卡片表达专业能力。
- 背景使用深海蓝渐变、网格纹理和电光蓝光晕，形成高端科技感。

### 5.3 核心数据背书

设计目标：快速建立可信度。

展示方式：

- 6 个玻璃拟态小卡片横向排列。
- 每个卡片只有一个大数字和一句解释。
- 数字使用蓝色，说明文字使用灰蓝。

推荐内容：

| 数据 | 文案 |
|---|---|
| 18年 | 技术 + 管理复合经验 |
| 90+ | 并发项目统筹 |
| 1000+ | 大型项目协同规模 |
| 亿级 | 大型直播高并发场景 |
| 70%+ | 线上问题反馈率降低 |
| 20%+ | 产研效能提升 |

设计注意：

所有数据正式上线前需确认可公开。若存在敏感风险，可改为“千万级”“大型复杂项目”等区间表达。

### 5.4 服务能力

设计目标：让客户快速判断“我是否需要这个服务”。

在服务能力前增加“典型项目痛点”模块，帮助访问者先自我识别问题：

1. 项目延期反复发生：目标、范围、依赖和关键路径不清，推进靠个人盯。
2. 跨团队协同低效：产品、研发、算法、测试、运营之间职责和节奏不一致。
3. 需求频繁变化：变更缺少评估、审批和影响说明，交付节奏被持续打断。
4. 上线质量不稳定：缺少准入准出、风险预案、发布门禁和复盘闭环。

布局：

- 使用纵向列表，而不是普通四宫格。
- 每项服务左侧有序号，右侧有箭头，强调“服务路径”和“可进入详情”的可能性。

服务内容：

1. AI Agent 平台项目管理
2. 战略 PMO 与项目治理
3. 产研流程体系 0→1→N 搭建
4. 效能度量与交付质量提升

设计理由：

纵向列表更适合专业服务官网，便于读者逐项扫描，也比图标卡片更稳重。

### 5.4.1 服务包设计

设计目标：把能力表达转化成客户能理解、能购买、能沟通的服务产品。

服务包：

| 服务包 | 说明 | 典型交付物 |
|---|---|---|
| 项目诊断 | 通过访谈、问卷和交付资料梳理，识别当前项目治理问题 | 项目风险清单、治理优先级、初步改进建议 |
| PMO 体系搭建 | 建立项目组合管理、里程碑、风险机制、版本门禁和复盘机制 | PMO 流程、模板、会议机制、看板框架 |
| AI Agent 项目交付陪跑 | 陪跑 Agent 产品从需求定义、研发推进、测试验收到上线发布 | 项目计划、需求拆解、风险台账、发布检查清单 |
| 效能看板建设 | 设计进度、质量、效率、风险和线上反馈指标 | 指标体系、项目健康度看板、复盘分析框架 |

### 5.5 代表案例

设计目标：用真实项目证明能力。

布局：

- 深海蓝底承接全站视觉，使用半透明玻璃卡片和顶部细色条建立案例区层次。
- 2×2 案例卡片。
- 每张卡片包含：项目类型、年份/阶段、标题、说明、结果数字。

案例优先级：

1. 字节跳动多业务线 PMO 体系搭建
2. 大型体育赛事直播稳定交付
3. AI Agent 平台从 0 到 1 发布
4. 多模态视频理解平台落地

视觉要点：

- 所有案例卡统一半透明深色玻璃背景，避免模块背景频繁跳色。
- 用顶部 5px 色条区分案例类型。
- 结果数字统一使用电光蓝，保证视觉识别稳定。

案例公开策略：

| 类型 | 使用条件 | 展示方式 |
|---|---|---|
| 公开版 | 已确认可公开客户名和数据 | 可展示“字节跳动”等实名客户 |
| 泛化版 | 未确认公开授权或涉及敏感业务数据 | 使用“头部内容平台”“大型体育赛事直播”“亿级峰值场景”“高准确率”等泛化表达 |

当前 V1.2 页面采用混合策略：保留已确认可公开的“字节跳动”案例，其余涉及赛事、峰值并发和准确率的数据采用泛化表达。

### 5.6 方法论

设计目标：证明润序科技不是“项目跟进”，而是“体系化治理”。

布局：

- 5 张步骤卡片。
- 每张卡片包含序号、标题、短说明。

五步路径：

1. 目标对齐
2. 项目拆解
3. 体系搭建
4. 数据度量
5. 持续改进

设计重点：

这个模块是从“个人能力”转向“公司方法论”的关键，应保持清晰、理性、可复制。

### 5.7 CMMI × 项目成功率诊断模型

设计目标：把咨询服务产品化，形成售前转化钩子。

布局：

- 深海蓝背景。
- 三张玻璃拟态诊断卡片：
  1. CMMI 成熟度映射
  2. 项目成功率评分
  3. 售前评测问卷

设计重点：

- `0-100` 作为视觉锚点。
- 用 L1-L5 表达过程成熟度。
- 明确“非官方 CMMI 评估”，避免认证误导。

当前 V1 已实现：

- `questionnaire.html` 独立自评页面。
- 问卷页采用深色诊断页布局：顶部说明 + 摘要卡 + 左侧评分面板 + 右侧分组问卷。
- 24 题问卷。
- 本地即时计算。
- 所有题目默认 0 分，页面打开即展示默认诊断结果。
- 全部完成后展示 0-100 分、CMMI 成熟度映射和治理建议。
- 结果区提供“带评测结果预约诊断”CTA。
- 映射 CMMI L1-L5 成熟度。
- 输出对应治理建议。

### 5.8 创始人背书

设计目标：将个人履历转化为公司可信资产。

内容：

- 范丽丽
- 高级项目经理 · TPM & PMO · PMP · CMMI3 评估师
- 18 年技术 + 管理复合经验
- AI Agent、AI 大模型、音视频、云游戏、Android 系统、ToB / ToC 产品交付、服务型 PMO 与战略 PMO
- 字节跳动集团奖、Spot Bonus、360 年度优秀个人、0-1 体系建设

设计注意：

不要做成简历列表，应强调“为什么这段经历能支撑公司服务”。

### 5.9 联系合作

设计目标：降低咨询门槛。

布局：

- 深海蓝页面背景中嵌入更强光晕的 CTA 玻璃卡片。
- 左侧：30 分钟项目诊断引导文案 + 邮箱。
- 右侧：飞书二维码。
- CTA 卡片使用深蓝渐变、电光蓝光晕和半透明边框，作为全站最强转化锚点。

咨询承诺表达：

- 用户可带当前项目目标、计划、风险或问卷结果来沟通。
- 诊断输出方向包括：项目风险清单、治理优先级、下一阶段改进路径。

交互：

- 邮箱点击唤起邮件。
- 二维码用于扫码添加飞书联系人。

## 6. 组件规范

### 6.1 按钮

主按钮：

- 背景：电光蓝渐变
- Hover：提亮 + 轻微上浮 + 蓝色阴影增强
- 圆角：100px
- 高度：54px

小按钮：

- 用于导航右侧 CTA
- 高度：42px

### 6.2 卡片

基础卡片：

- 圆角：22-28px
- 边框：`rgba(255, 255, 255, .13)`
- 背景：`rgba(255, 255, 255, .075)`
- 质感：`backdrop-filter: blur(18px)`，配合内高光和深色阴影

深色卡片：

- 背景：深海蓝 / 深蓝渐变，用于 Hero 视觉、问卷评分面板和最终 CTA 卡片
- 边框：半透明白
- 强调色：荧光绿

### 6.3 标题层级

| 层级 | 使用场景 | 字号建议 |
|---|---|---|
| H1 | Hero 主标题 | 48-82px |
| H2 | 模块标题 | 40-66px |
| H3 | 卡片标题 | 24-40px |
| Body | 正文说明 | 16-18px |
| Caption | 标签、元信息 | 10-13px |

## 8. 交互设计

当前 V1 使用轻交互：

1. 平滑滚动导航。
2. CTA hover 上浮。
3. 服务列表箭头 hover 位移。
4. 邮箱 mailto。
5. 飞书二维码静态展示。

暂不引入复杂动画，避免影响性能和正式上线稳定性。

问卷状态：

| 状态 | 页面表现 |
|---|---|
| 初始状态 | 所有题目默认 0 分，侧栏显示 0 分、极高风险 · CMMI L1 |
| 调整中 | 用户修改任一题目后，实时更新项目成功率得分、成熟度等级和治理建议 |
| 转化 | 始终展示“带评测结果预约诊断”按钮，跳转至联系区 |

## 8.1 SEO 与分享基础项

V1.2 增加以下上线基础要求：

1. 首页 title、description、keywords。
2. 问卷页 title、description。
3. Open Graph title、description、type。
4. favicon。
5. 后续部署正式域名后补充 canonical URL 和分享图。

## 9. 可访问性要求

1. 页面只保留一个 H1。
2. 图片必须有 alt。
3. 文字与背景对比度要足够。
4. CTA 可点击面积不小于 42px 高。
5. 移动端无横向滚动。
